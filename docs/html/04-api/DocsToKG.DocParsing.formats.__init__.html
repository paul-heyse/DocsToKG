1. Module: init

This reference documents the DocsToKG module DocsToKG.DocParsing.formats.

1. Overview

DocsToKG.DocParsing.formats gathers the canonical DocParsing schemas, validation helpers, and Docling serializer utilities into a single import surface. Pydantic v2 is a hard requirement; the module instantiates real models at import time and raises actionable errors if the dependency is missing.

Key features:

  • Strict Pydantic models for chunk JSONL rows and embedding vector rows.

  • Co-located schema-version metadata (SchemaKind, SchemaVersion, validate_schema_version, ensure_chunk_schema).

  • Convenience validators (validate_chunk_row, validate_vector_row) that wrap model construction and dimension checks.

  • Markdown-aware Docling serializer providers for picture/table enrichment.

Dependencies:

  • pydantic>=2,<3 for model parsing and validation.

  • docling_core for serializer base classes and helpers.

2. Schema Metadata

  • SchemaKind enumerates the "chunk" and "vector" schema families.

  • SchemaVersion aliases the schema version strings used in manifests.

  • CHUNK_SCHEMA_VERSION / VECTOR_SCHEMA_VERSION expose the canonical versions emitted by DocParsing stages.

  • COMPATIBLE_CHUNK_VERSIONS / COMPATIBLE_VECTOR_VERSIONS list historically supported versions.

  • get_default_schema_version(kind) returns the canonical version for a schema kind.

  • get_compatible_versions(kind) lists all compatible versions.

  • validate_schema_version(version, kind, ...) raises ValueError if version is incompatible.

  • ensure_chunk_schema(rec, ...) backfills or validates chunk schema versions.

3. Models

  • ProvenanceMetadata stores parser provenance for each chunk.

  • ChunkRow validates chunk JSONL rows and enforces schema versions.

  • BM25Vector, SPLADEVector, and DenseVector model individual vector payload types.

  • VectorRow aggregates sparse and dense vectors plus optional metadata.

4. Validation Helpers

  • validate_chunk_row(row) constructs ChunkRow instances and raises ValueError on invalid input.

  • _pydantic_validate_vector_row(row) builds VectorRow objects directly (internal helper).

  • validate_vector_row(row, expected_dimension=None) wraps the internal helper and enforces dense vector dimensionality.

5. Serializer Utilities

  • get_docling_version() reports the installed Docling version or "unknown".

  • CaptionPlusAnnotationPictureSerializer enriches Docling picture items with captions, annotations, and confidences.

  • RichSerializerProvider constructs Markdown-aware serializer pipelines for chunking.
